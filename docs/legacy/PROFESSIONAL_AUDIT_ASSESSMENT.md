#### **A. MATERIALITY CALCULATION (ISA 320)** - Score: 10/10 ✅


#### **B. RISK ASSESSMENT FORMULAS** - Score: 8/10 ✅

**Audit Risk Model (1-5 Scale):**

```
INHERENT RISK FORMULA:
IR = (Complexity × 0.35 + Judgment × 0.35 + Prior Exceptions × 0.30)

Example: Revenue (ISA 240 high-risk area)
├─ Complexity: 4/5 (multiple methods, cutoff issues)
├─ Judgment: 5/5 (revenue estimates, contracts)
├─ Prior Exceptions: 2/5 (some prior issues)
└─ IR = (4×0.35 + 5×0.35 + 2×0.30) = 3.9/5 = HIGH ⚠️

CONTROL RISK FORMULA:
CR = (Design Effectiveness × 0.5 + Operating Effectiveness × 0.5)

Example: Revenue approval control
├─ Design: Effective for authorization (controls exist)
├─ Operating: 1 deviation in 50 items tested = 98% effective
└─ CR = (1.0×0.5 + 0.98×0.5) = 0.99 ≈ 1/5 = LOW ✅

DETECTION RISK CALCULATION:
DR = Acceptable DR Target / (IR × CR)

Example:
├─ Acceptable DR Target: 5% (conservative)
├─ IR: 0.78, CR: 0.20
└─ DR = 0.05 / (0.78 × 0.20) = 0.32 (32%) → REQUIRES EXTENSIVE TESTING
```

**Professional Application**: Risk-based sample sizing
**Accuracy**: Mathematically sound
**Limitation**: No entity-level rolldown (firm-wide risk aggregation)

#### **C. SAMPLE SIZE DETERMINATION (ISA 530)** - Score: 8.5/10 ✅

**Statistical Formula Implemented:**

```
STATISTICAL SAMPLE SIZE:
n = (Z² × p(1-p)) / E²

Where:
├─ Z = Confidence level factor
│  ├─ 68% confidence: Z = 1.0
│  ├─ 90% confidence: Z = 1.645
│  └─ 97% confidence: Z = 2.17
├─ p = Expected error rate (1-5% depending on risk)
├─ E = Acceptable error margin
└─ n = Calculated sample size

EXAMPLE: High-Risk Area
├─ Population: 1,200 invoices
├─ Materiality: $50,000 OM, $37,500 PM
├─ Risk level: HIGH (≤2% DR required)
├─ Calculation:
│  ├─ Z = 2.17 (97% confidence for high risk)
│  ├─ p = 0.03 (3% expected error)
│  ├─ E = $37,500 / 1,200 = $31.25
│  └─ n = (2.17² × 0.03 × 0.97) / (31.25)² = 68 items
├─ Plus: 100% testing for items > $50,000 (high-value items)
├─ Plus: Stratified by region (3 strata)
└─ FINAL SAMPLE: 75-85 items (highly materiality-driven)

RISK MULTIPLIERS APPLIED:
├─ High Risk: 3.0× (low detection risk acceptable)
├─ Medium Risk: 2.0×
├─ Low Risk: 1.0×

ACCOUNT-SPECIFIC MULTIPLIERS:
├─ Revenue: 1.2× (high risk, ISA 240)
├─ Receivables: 1.3× (cutoff, aging risks)
├─ Inventory: 1.2× (valuation, obsolescence)
├─ Provisions: 1.4× (judgment-intensive)
├─ Payables: 0.9× (less material, stable)
└─ Cash: 0.8× (well-controlled, low risk)
```

**Professional Quality**: ISA 530 compliant
**Field Implementation**: Auditors can override with justification
**Verification**: Calculations are auditable and explainable

#### **D. EXCEPTION EVALUATION & EXTRAPOLATION (ISA 530)** - Score: 8.5/10 ✅

**Exception Projection Formula:**

```
QUANTITATIVE EVALUATION:

ExceptionRate = Exceptions Found / Sample Size
ProjectedPopulation = ExceptionRate × Total Population

Example: Cutoff Testing
├─ Exceptions found: 2 invoices (one side of cutoff)
├─ Sample size: 75 items
├─ Total population: 1,200 transactions
├─ ExceptionRate = 2 / 75 = 2.67%
├─ ProjectedMisstatement = 0.0267 × 1,200 = 32 items
├─ If avg amount: $2,000
└─ Projected Impact: 32 × $2,000 = $64,000 ⚠️

MATERIALITY IMPACT:
├─ % of PM: $64,000 / $37,500 = 170.7% ❌ EXCEEDS MATERIALITY
├─ % of OM: $64,000 / $50,000 = 128% ❌ EXCEEDS THRESHOLD
└─ CONCLUSION: NOT ACCEPTABLE without further investigation/adjustment

QUALITATIVE FACTORS ASSESSED:
├─ Nature: Systematic (cutoff) vs Isolated (data entry error)?
├─ Pervasiveness: One-off or pattern of carelessness?
├─ Fraud indicators: Intentional or unintentional error?
├─ Root cause: Process failure or control breakdown?
├─ Implications: Do other controls compensate?
└─ Decision: Request adjustment or conduct extended testing
```
