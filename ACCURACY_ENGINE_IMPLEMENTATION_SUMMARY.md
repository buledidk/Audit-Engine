## 🚀 How to Use

### Quick Start (3 steps)

**Step 1**: Add to server (1 line)
```javascript
app.use('/api/accuracy', accuracyRoutes);
```

**Step 2**: Update .env
```bash
ENABLE_ACCURACY_ENHANCEMENTS=true
```

**Step 3**: Call API
```bash
curl -X POST http://localhost:3000/api/accuracy/enhance \
  -H "Content-Type: application/json" \
  -d '{"auditData": {...}}'
```

## 🔧 Configuration & Customization

### Enable/Disable Enhancements
```javascript
const engine = new AuditAccuracyEnhancementEngine({
  enableAnomalyDetection: true,  // Toggle any
  enableFraudDetection: true,
  enableConsensus: false,        // Disable if not needed
  // ... etc
});
```

### Adjust Thresholds
```javascript
const engine = new AuditAccuracyEnhancementEngine({
  consensusThreshold: 0.85,      // Higher = stricter consensus
  confidenceThreshold: 0.80,     // Higher = only high-confidence
  anomalyStdDev: 3.0             // Higher = fewer anomalies flagged
});
```

### Add Custom Enhancements
Each enhancement module can be extended with custom logic:

```javascript
class CustomFraudEngine extends FraudPatternRecognitionEngine {
  analyzeFraudRisks(auditData) {
    const baseResult = super.analyzeFraudRisks(auditData);
    // Add custom detection logic
    return baseResult;
  }
}
```

```bash

```

### Slow Performance?
```javascript
// Disable non-critical enhancements
const fastEngine = new AuditAccuracyEnhancementEngine({
  enableBlockchainEvidence: false,
  enableContinuousAssurance: false
});
```

### Low Confidence Scores?
```bash
# Check data quality first
POST /api/accuracy/data-quality
# Then improve data before re-running
```

### Memory Issues?
```javascript
// Process in batches
const batches = chunkArray(largeDataset, 500);
for (const batch of batches) {
  await engine.enhanceAuditAccuracy({ transactions: batch });
}
```
